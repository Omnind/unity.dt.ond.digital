{"Title":"Logic Simulator (TWINCAT_ADS)","Text":"<size=16><b>Logic Simulator</b></size>\r\n\r\nA <b>Logic Simulator</b> handles all communication between Unity and an external controller for all Logic Components that are parented under the component.\r\n\r\nTo create a <b>Logic Simulator</b>, go to <i>Logic > Add Logic Simulator</i>. \r\nYou can have this component be assigned to an existing GameObject by selecting it in the hierarchy before adding the component, or you can automatically create a new GameObject by not having anything selected at all. \r\n\r\nTo setup the PRElogic Simulator you need to follow these steps:\r\n<b>[Simulator Settings]</b>\r\n  - Fill in the <b>Simulator Name</b> (any).\r\n  - Check <b>Start in Playmode</b>.\r\n\r\n<b>[Gateway Settings]</b>\r\n  - The <b>Adapter Target</b> needs to be set to \"TWINCAT_ADS\".\r\n  - Fill in the <b>XmlFilePath</b>. Make sure it ends with <b>\".xml\"</b>.\r\n  - Fill in the <b>Server Name</b>.\r\n  - Fill in the <b>IPAdress</b>. This needs to be the <b>AMS Net ID</b> of the PLC. Add a <b>colon</b> behind it followd by the PLC <b>port</b> (default :851). \r\n\r\n<b>[Configuration Export Settings]</b>\r\n  - Unfold <b>Configuration Settings</b>. Insert the PLC Project Folder into <b>Project Export Folder</b>\r\n\r\nIf all LogicController have been assigned (shown under <i>Component Settings</i>), under <b>Policy Settings</b> click <b>Export Policy to XML</b>.\r\nLastly, under <b>Configuration Export Settings</b> click <b>Export Configuration Files</b>.\r\n\r\nIf no errors have occured, everything should now have been setup properly in Unity.\r","DocumentationURL":"https://unit040.atlassian.net/wiki/spaces/PUD/pages/280987237/Logic+Simulator","SubComponents":"","Locked":true}
